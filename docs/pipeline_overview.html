Phase 2 — 분류 · 정제 · 인과 분석 (외부 배치)
4

뉴스 분류 (Haiku LLM)

14개 V2 토픽으로 분류 + 13키 자산영향도 벡터 산출. Financial Filter로 비금융 기사 제거.

~$100/월 14개 토픽
analyze/news_classifier.py
5a

중복 제거 (Dedupe)

같은 사건을 여러 매체가 보도하면 수십 건의 유사 기사가 생깁니다. 이를 정리하는 과정입니다.

① URL 정규화: 쿼리 파라미터(?ref=naver 등)를 제거하여 같은 기사의 다른 URL을 통일합니다.
② 강한 중복: 제목이 90% 이상 일치하면 하나만 남기고 제거합니다.
③ 약한 중복 (대표 기사 선정): 같은 사건을 다루는 기사들을 그룹으로 묶고, 그 중 가장 신뢰도 높은 매체의 기사를 "대표 기사(primary)"로 선정합니다. 나머지는 중복으로 처리합니다.

예: "한은 금리 동결" 기사 30건 → 대표 1건 + 교차보도 기록
core/dedupe.py
5b

이벤트 클러스터링 (Event Clustering)

서로 관련된 토픽의 기사들을 하나의 "이벤트 그룹"으로 묶는 과정입니다.

예시: "연준 금리 동결" 기사(통화정책)와 "미국채 금리 하락" 기사(금리/채권)는 별도 토픽이지만 같은 이벤트(연준 결정)에서 비롯됩니다. 이런 기사들을 하나의 이벤트 그룹으로 묶어 중복 분석을 방지합니다.

토픽 이웃 관계: 금리↔통화정책, 환율↔달러유동성, 에너지↔지정학 등 8개 그룹으로 교차 매핑됩니다.

같은 이벤트의 다른 시각 → 1그룹
core/dedupe.py → TOPIC_NEIGHBORS
5c

중요도 점수 산출 (Salience Scoring)

각 기사가 "시장에 얼마나 중요한 뉴스인지"를 0~1 점수로 평가합니다.

산출 공식:

매체 등급 (30%)TIER1(Reuters,연합뉴스)=1.0 / TIER2(매경,한경)=0.7 / TIER3(기타)=0.3
기사 강도 (25%)LLM이 판단한 시장 영향 강도 (1~10) 정규화
교차 보도 (25%)같은 이벤트를 몇 개 매체가 보도했는지 (많을수록 높음)
BM 이상치 (20%)해당 날짜에 벤치마크 수익률이 급변(z>1.5)했으면 1.0

결과: 점수가 높은 기사가 debate 입력에 우선 선택됩니다.

예: Reuters "이란 전쟁" (salience 0.95) > 블로그 "환율 전망" (salience 0.30)
core/salience.py
6a

뉴스 샘플링 (Stratified Sampling)

전체 뉴스(수천~수만 건)를 모두 분석하면 비용이 과다하므로, 토픽별로 균등하게 샘플링합니다.

방법: 각 토픽에서 최소 10건을 먼저 확보한 뒤, 나머지는 salience 높은 순으로 채웁니다. 총 300~500건.

14개 토픽 × 최소 10건 + salience 상위
6b

엔티티 추출 (Entity Extraction)

"엔티티"란 뉴스에 등장하는 핵심 개념/사건/지표를 말합니다. LLM(Haiku)이 각 기사에서 주요 엔티티를 뽑아냅니다.

예시:

"이란이 호르무즈 해협 봉쇄를 예고했다"→ 이란, 호르무즈_봉쇄, 유가_급등
"연준이 금리를 동결했다"→ 연준, 금리_동결, 달러_약세
"엔비디아 실적이 예상치를 상회했다"→ 엔비디아, AI_반도체, 기술주_강세
Haiku LLM 기사 1건 → 엔티티 2~5개
6c

인과관계 추론 (Causal Inference)

추출된 엔티티들 사이에 "A가 B를 야기했다"는 인과 관계를 LLM(Sonnet)이 추론합니다.

예시 인과 경로:

호르무즈_봉쇄 → 유가_급등 → 인플레_재점화 → 금리_인상_우려신뢰도 0.92
연준_금리동결 → 달러_약세 → 원화_강세 → 수출주_부담신뢰도 0.85
엔비디아_호실적 → AI_투자확대 → 반도체_수출호조 → KOSPI_상승신뢰도 0.78

이 인과 경로들이 debate 시 에이전트와 Opus에게 "시장이 왜 이렇게 움직였는지"의 논리 구조를 제공합니다.

Sonnet LLM 엣지 = 인과관계 1개
6d

시간 지식 그래프 (Temporal Knowledge Graph)

매월 추출된 엔티티와 인과관계를 시간축이 있는 그래프로 누적 관리합니다.

자동 관리 규칙:

Decay (감쇠)오래된 인과관계의 신뢰도를 시간이 지나면 자동 감소
Merge (병합)같은 인과관계가 반복 확인되면 신뢰도 상승
Prune (가지치기)신뢰도가 너무 낮아진 관계는 그래프에서 제거

결과물: 노드(엔티티) + 엣지(인과관계) + 전이경로(A→B→C 체인) 형태의 JSON 파일.

예: 131노드, 140엣지, 9개 전이경로
analyze/graph_rag.py → data/insight_graph/*.json

report/report_store.py
17

Client 조회

운용보고(매크로): 시장 코멘트 + 출처 + 관련 지표 차트. 운용보고(펀드): 펀드 코멘트. draft/warning/evidence raw는 미노출.

운용보고(매크로) 운용보고(펀드)
매크로 코멘트 vs 펀드 코멘트 비교

매크로 코멘트

  • 입력: 뉴스 + GraphRAG + 지표 + 블로그
  • 생성: 4인 debate → Opus 종합
  • 내용: 글로벌/국내 거시환경 분석
  • 저장: _market.draft.json
  • 서비스: debate_service.py
  • 비용: ~$0.34/회
  • 빈도: 월 1회 또는 분기 1회
  • 펀드 무관: 시장 전체 관점

펀드 코멘트

  • 입력: 시장 debate 결과 + PA + 보유 + 거래
  • 생성: comment_engine → Opus
  • 내용: 펀드별 운용경과/전망/매니저코멘트
  • 저장: {fund_code}.draft.json
  • 서비스: fund_comment_service.py
  • 비용: ~$0.50/펀드
  • 빈도: 펀드별 개별 생성
  • 미편입 자산군 자동 제외
