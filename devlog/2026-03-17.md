### 2.3 원인 추적 과정

1. **통화 매핑 확인** — R/Python 동일 (`solution.universe_non_derivative`)
2. **ECOS 환율 확인** — R/Python 동일 (return 차이 0건)
3. **R 원본 PA 파이프라인 실행** (debug_pa_original.R)
   - R 중간 데이터(r_sec, weight_PA) = Python과 완벽 일치
   - → 차이는 경로의존 누적기여도 계산에서 발생
4. **R의 `수익률(FX_제외)` 계산 방식 발견** (line 561):
   ```
   수익률(FX_제외) = sum(총손익금액_당일_FX_adjust) / sum(조정_평가시가평가액)
   ```
   - `총손익금액_당일_FX_adjust = 총손익금액 - 환산_adjust`
   - `환산_adjust = 시가평가액(T-1) × r_FX × (1 + r_sec)`
5. **근본 원인**: 종목 첫 등장일에 `시가평가액(T-1) = 0` → `환산_adjust = 0` → FX 미제거
   - Python(기존): `r_sec = (1+R)/(1+r_FX)-1` — 수식 기반, 항상 FX 제거
   - R: 금액 기반 — T-1 보유=0이면 환효과=0 (실제 환노출 기간 기준)

### 2.4 R 로직의 근본 목적

**"실제로 환율에 노출된 기간(T-1 보유)에 대해서만 환효과 인식"**
- 신규 매수일: 환노출 기간 = 0 → 환효과 = 0 (합리적)
- Python 기존: 수학적으로 항상 FX 분리 → 환노출 없는 날에도 FX 차감 (과대)

### 2.5 수정 내용

`modules/data_loader.py` — `compute_single_port_pa()` FX split 섹션:
```python
# 기존 (수식 기반)
수익률_사용 = (1 + 종목별수익률) / (1 + return_USDKRW) - 1

# 변경 (R 동일, 금액 기반)
환산_adjust = 시가평가액_T1 * return_USDKRW * (1 + r_sec)
수익률_사용 = (총손익금액 - FX효과금액) / 조정_평가시가평가액
```
