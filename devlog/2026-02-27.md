### 2.5 Brinson 컬럼명 버그 수정 (핵심)

**문제:** `compute_brinson_attribution()`에서 보유종목 DataFrame의 컬럼명을 잘못 참조.

| 위치 | 버그 (한국어 별칭) | 수정 (실제 컬럼명) |
|------|-------------------|-------------------|
| line 1079 | `holdings['종목코드']` | `holdings['ITEM_CD']` |
| line 1187 | `holdings['종목코드']`, `['종목명']` | `holdings['ITEM_CD']`, `['ITEM_NM']` |

**원인:** `load_fund_holdings_classified()`는 DB 원본 컬럼명(`ITEM_CD`, `ITEM_NM`)을 반환하지만, `compute_brinson_attribution()`은 한국어 별칭(`종목코드`, `종목명`)으로 참조. `KeyError` 발생 → `try/except: pass`로 무시 → 목업 fallback.

**증상:** 08K88 등 모든 펀드에서 Tab 3 Brinson PA가 "⚠️ 목업 데이터"로 표시됨.

### 2.6 에러 가시성 개선

`prototype.py` Tab 3의 `try/except: pass` → 실패 원인을 캡션에 표시:

```python
# Before
except Exception:
    pass

# After
except Exception as e:
    _brinson_fail_reason = f"DB 조회 오류: {e}"

# 캡션에 원인 표시
st.caption(f"⚠️ 목업 데이터 — {_brinson_fail_reason}")
```

`compute_brinson_attribution()`에도 디버그 print 추가.
