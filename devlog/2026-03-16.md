## 3. 기간수익률 DT 일치

### 3.1 문제

Python 기존: `timedelta(days=30)` → 3/15 기준 1M = 2/13
DT 시스템: `relativedelta(months=1)` → 3/15 기준 1M = 2/15

모든 기간(1M/3M/6M/1Y)에서 0.01~0.05%p 차이 발생.

### 3.2 해결

```python
from dateutil.relativedelta import relativedelta
'1M': _end_dt - relativedelta(months=1),
'3M': _end_dt - relativedelta(months=3),
'6M': _end_dt - relativedelta(months=6),
'1Y': _end_dt - relativedelta(years=1),
```

### 3.3 검증 (4JM12, 기준일 2026-03-13)

| 기간 | Python | DT | diff |
|------|--------|------|------|
| 1D | 0.2101% | 0.2101% | 0.0000 |
| 1W | -0.1447% | -0.1447% | 0.0000 |
| 1M | 1.2375% | 1.2375% | 0.0000 |
| 3M | 2.1619% | 2.1619% | 0.0000 |
| 6M | 5.4519% | 5.4519% | 0.0000 |
| YTD | 3.1268% | 3.1268% | 0.0000 |

## 4. 설정후 수익률 보정

### 4.1 문제

4JM12 DB 첫 MOD_STPR = 1998.62 → 설정후 수익률 = 30.50%
시스템 설정후 수익률 = 32.3484% (기준가 1970.76 기준)

DWPM10040의 `OPNG_NEXT_FEE_BFSB_ERN_RT` = 30.50% (MOD_STPR 첫값 기준, 보수차감전)
시스템은 별도 기준가(1970.76)로 계산.

### 4.2 해결

```python
_FUND_INCEPTION_BASE = {'4JM12': 1970.76}
```

- 설정후 수익률: `_FUND_INCEPTION_BASE.get(fund, nav_data[0])` 기준
- 메트릭 카드, 누적수익률 차트도 동일 기준가 사용
- BM은 DT MOD_STPR 첫값(1000) 기준 변동 없음

### 4.3 검증

| 항목 | Python | 시스템 |
|------|--------|--------|
| 펀드 설정후 | 32.35% | 32.3484% |
| BM 설정후 | 31.54% | 31.5378% |
