## 3. 무위험수익률 소스 비교

### 3.1 ECOS API → KIS CD Index 교체

R Shiny는 `ecos` R 패키지로 매번 API 호출. Python에서는 DB 소스로 교체.
- SCIP DB에 ECOS source 없음 (back_source 13개 중 ECOS 미포함)
- KIS CD Index (dataset_id=194, dataseries_id=33) 선택

### 3.2 세 소스 비교 (08N81 기간)

| 소스 | dataset_id | 시작 | 누적 연율화 | ECOS 대비 |
|------|-----------|------|-----------|----------|
| ECOS API | - | 1999~ | 2.7937% | 기준 |
| KIS CD Index 총수익 | 194 | 2015~ | 2.7936% | **0.01bp** |
| KAP CD 총수익지수 | 300 | 2001~ | 2.6757% | **11.8bp** |

KIS 채택 이유: ECOS 대비 0.01bp 오차, blob에 `totRtnIndex` 직접 제공, API 호출 불필요.

## 4. DB 컬럼명 수정

DWCI10220 실제 컬럼명이 소문자:
- `std_dt` (기존 코드: `CAL_DT`)
- `hldy_yn` (기존 코드: `HOLI_FG`, 값: 'Y'/'N' — '0'/'1' 아님)
- `day_ds_cd`

`load_holiday_calendar()`: `AS CAL_DT`, `AS HOLI_FG` alias 처리.
`get_business_days()`: 'N'/'0' 양쪽 호환 처리 추가.
