## 3. 옵션 평가

### Option A — defer 유지 (현행)
- 장점: contract 위반 0, false positive 0
- 단점: `달러` 노드 영원히 entity 후보 미진입

### Option B — context split rule
- 분류기 결과(`_asset_impact_vector` 또는 `primary_topic`)에 따라 다른 taxonomy 매핑
- 장점: 의미별 정확 매핑 가능
- 단점:
  - 분류기 신뢰도 의존성 (Haiku 분류 오류율 누적)
  - entity_builder가 article-level 정보를 다시 봐야 함 (모듈 경계 위반)
  - "노드 → taxonomy" 1:N 매핑이라 entity_id 충돌 위험
- evidence 부족

### Option C — single alias 고정
- (C-1) `달러 → 환율_FX`: 정확도 52% (1878/3625). 절반 wrong.
- (C-2) `달러 → 달러_글로벌유동성`: 정확도 **1%** (38/3625). 거의 다 wrong.
- 어느 쪽이든 false positive 폭증 위험

---

## 4. 결론

**Option A (defer 유지) 채택**.

근거:
- 표본 분석 + classifier 분포 모두 `달러`는 단독 의미가 없는 다의어 핵심 사례
- "많이 잡는 정책"보다 "덜 틀리는 정책" 우선 원칙 (작업지시서 §Step 2)
- v11/v13 contract: ambiguous phrase 억지 매핑 금지 → 본 사례에 정확히 해당
- false positive 1700+ 건 폭증 위험을 감수할 만한 entity 가치 없음

**다음 라운드 재검토 조건**:
- 분류기 정확도 별도 측정 → 95%+ 도달 시 Option B 재검토
- `달러_글로벌유동성` 자체에 더 많은 직접 alias 추가 후 정책 재논의
- 별도 entity 정책 (예: `달러` → 다중 entity로 분해) 신설 가능성

## 5. 영향

이번 배치에서 `달러` 단독은 **alias 미추가**.
graph node `달러`는 계속 entity 후보 풀에서 탈락 (taxonomy gate miss).
defer 사유는 `entity_alias_diversity_review.md`에도 반영.
